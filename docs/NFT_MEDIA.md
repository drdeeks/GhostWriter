# NFT Media Guide

This guide explains the pipeline for generating NFT metadata and images in Ghost Writer, and how you can customize the appearance of the NFTs.

## 🎨 Metadata Generation

When an NFT is minted, its metadata is not stored directly on the blockchain. Instead, the `GhostWriterNFT.sol` contract returns a URI pointing to an API route: `/api/nft/[tokenId]`. This route is responsible for generating the metadata for a given NFT on the fly.

The metadata includes:
- **`name`**: The title of the NFT (e.g., "Ghost Writer Contributor #123").
- **`description`**: A description of the story and the user's contribution.
- **`image`**: A URL pointing to the NFT's image, generated by the `/api/nft/[tokenId]/image` route.
- **`attributes`**: An array of traits, such as the story category, word type, and date of contribution.

## 🖼️ Image Generation

The NFT images are dynamically generated as SVG files by the `/api/nft/[tokenId]/image` route. This allows for a high degree of customization and ensures that each NFT's image is unique.

The SVG can include:
- The full story text with the contributor's word highlighted.
- The story's title and category.
- The contributor's Farcaster username or wallet address.
- A custom background image.

### Customizing NFT Backgrounds

You can customize the background of the NFT images by providing either local or remote image URLs.

#### Local Backgrounds
1.  Place your background images in the `public/nft-backgrounds/` directory.
2.  Set the `NFT_BACKGROUND_BASE_URL_LOCAL` environment variable to `/nft-backgrounds`.
3.  Use the `NFT_BACKGROUND_CATEGORY_MAP` variable to map story categories to your image filenames. For example:
    ```
    NFT_BACKGROUND_CATEGORY_MAP='{"adventure": "adventure-bg.png", "sci-fi": "scifi-bg.jpg"}'
    ```

#### Remote Backgrounds
1.  Host your background images on a service like IPFS or a CDN.
2.  Set the `NFT_BACKGROUND_BASE_URL_REMOTE` environment variable to the base URL of your image host.
3.  Use the `NFT_BACKGROUND_CATEGORY_MAP` as described above.

#### Fallback Order
When generating an NFT image, the application will look for a background in the following order:
1.  Local background image
2.  Remote background image
3.  The default procedurally generated gradient

This allows you to provide a rich set of custom backgrounds while still having a graceful fallback for categories that do not have a mapped background.
